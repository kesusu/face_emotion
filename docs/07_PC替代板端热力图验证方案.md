1. **在PC上用MediaPipe生成全部4,574张图片的热力图(npy)**

但在执行前，需要验证：**PC的MediaPipe和板端的fiboaisdk输出是否一致？**

## 四、如果验证通过，我打算做的优化方案

### 方案A: 仅替换数据源（保守）

保持所有参数不变，只是把热力图的来源从板端换成PC端MediaPipe。

```
改动量: 最小 (只换数据来源)
预期收益: 无 (如果兼容的话准确率不变)
目的: 解除对板端的依赖，可以在PC GPU上快速迭代
```

### 方案B: 热力图参数优化（推荐）

同时修改以下参数来提升模型能力：

| 参数 | 当前值 | 改成值 | 原因 |
|------|--------|--------|------|
| `HEATMAP_SIZE` | **64×64** | **128×128** | 保留更多FACS细节(AU4眉压/AU9鼻皱) |
| `GAUSS_SIGMA` | **1.5** | **1.0** | 减少模糊，相邻关键点区分更清楚 |
| nose权重 | **1.0** | **2.5** | 提升厌恶/愤怒的核心特征区域 |

```
改动量: 改3行代码 + 全量重新生成热力图 + 重训CNN
预期收益: CNN总准确率 40% → 45-52% (+5~12pp)
主要受益类别: 厌恶(+5~8pp), 悲伤(+5~10pp), 恐惧(+3~5pp)
PC GPU重训时间: 约15-20分钟 (128×128输入)
```

### 方案C: 多Channel架构升级（激进）

把当前的单通道Ch2拆分为独立通道：

```
当前:  3通道 [全点xy, z深度, 区域混合]
改为:  5-6通道 [眉毛单独, 眼睛单独, 鼻子单独, 嘴巴单独, 轮廓, z深度]

改动量: 重写热力图函数 + 修改CNN输入层 + 重训
预期收益: +3~6pp (让CNN能学各区域的独立模式)
风险: 更复杂，可能过拟合(数据量不够时)
```

### 推荐执行顺序

```
第一步: 你帮我做上面的对比验证 → 确认兼容性
         ↓ 通过
第二步: 执行方案B (128×128 + sigma 1.0 + nose权重↑) 
         → PC GPU重训 → 看效果
         ↓ 效果好
第三步: 补充悲伤/惊讶的训练数据(之前分析的缺口大的类)
         → 再训一轮 → 得到最终最优模型
         ↓
第四步: 推回板端 + 跑Ensemble融合
```
