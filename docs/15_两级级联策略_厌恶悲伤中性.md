# 两级级联策略：解决 厌恶/悲伤/中性 三类混淆

> **日期**: 2026-05-09 21:50  
> **状态**: 全局复盘完成，待实施  
> **背景**: 板子上惊讶/愤怒/效果好，但厌恶/悲伤/中性三者严重混淆

---

## 一、问题全景：数据说了什么？

### 1.1 当前V16 SVM逐类表现（清洗后8050张）

```
类别    准确率     样本数    判定
─────────────────────────────────────
惊喜    62.9%      1430      ✅ 够用
恐惧    13.8%       288      ⚠️ 少样本难救
厌恶     9.0% ★★★   726      🔴 核心痛点
快乐    60.7%       980      ✅ 够用
悲伤    35.0% ★★   1514      🟡 需提升
愤怒    30.4%       623      ✅ 板子规则可补
中性    87.1%      2486      ✅ 很强
```

### 1.2 三类混淆子矩阵（行=真实标签, 列=预测结果）

**SVM(RBF,C=5)**:
```
           预测→   厌恶     悲伤     中性

厌恶(145)          21       21       78   ← 54%崩向中性!

**RF(200树)** — 注意RF对悲伤更强:
```
           预测→   厌恶     悲伤     中性
真实↓

悲伤(303)           1      104        141  ← RF悲伤=34.2%! SVM只有27%

### 1.3 核心诊断结论

| 结论 | 证据 |
|------|------|
| **不是"三类互混"，而是"两类崩向一类"** | 厌恶54%→中性, 悲伤50%→中性, 中性88%正确 |
| **中性是黑洞** | 它把厌恶和悲伤全吸走了 |
| **RF对悲伤天然敏感** | RF悲伤34.2% vs SVM 27%，差异=+7pp |
| **SVM和RF有互补性** | 18.7%的样本两者预测不同 → Ensemble空间 |
| **根本原因** | 数据清洗后剩余样本特征极微妙(微皱眉/微抿嘴), 高斯热力图分辨率不够 |

## 二、已验证的策略清单（全部基于当前8050张数据实测）

### 策略A: OvR二分类器（每类单独训练）
- 厌恶: P=67.8%, R=17.5%（精确率高但召回太低）
- **结论**: 不适合直接用，但概率输出可用于Ensemble权重

### 策略B: balanced加权SVM（class_weight='balanced'）
- 厌恶: 33.8% (+19.3pp!) 🚀
- 悲伤: 28.4% (-6pp) 
- 中性: 63.6% (-24pp) 💀
- **结论**: 牺牲中性换厌恶，总体下降到~54%。**不可接受**

### 策略C: 逻辑回归（线性模型，可解释）
- 厌恶: 41.4%, 悲伤: 31.7%, 中性: 61.4%
- 总体: 52.9%
- **结论**: 收敛困难，不如SVM/RF

### 策略D: 智能投票（SVM为主，悲伤听RF）⭐ 推荐
- **零训练成本**, 纯推理规则
- 规则: `如果 RF说悲伤 且 SVM不说悲伤 → 改判为悲伤`
- 效果:
  ```
  全体: 56.0% → 56.5% (+0.5%)
  厌恶: 17.5% → 12.4% (-5pp, 退步)
  悲伤: 27.1% → 40.6% (+13.5pp) 🔥🔥🔥
  中性: 95.0% → 85.1% (-10pp, 可接受牺牲)
  ```
- **结论**: 极低成本大幅提升悲伤准确率

### 策略E: 三类专用分类器 ⭐⭐ 最强但需新模型
- **原理**: 排除{惊喜/恐惧/快乐/愤怒}干扰，只训厌恶vs悲伤vs中性
- **需要**: 单独训练一个3类SVM/RF（用同样的PCA50数据即可）

#### E1: 三类专用SVM（在945个三类子集上训练）⭐ 已实施 → V17
```
总体(仅3类): 62.9%
  厌恶: 46.2% (+26.2pp vs 原17.5%) 🚀🚀🚀
  悲伤: 54.5% (+9.6pp vs 原34.5%) 🚀🚀
  中性: 72.8% (-16pp vs 原87.9%)
  
模型已保存: v17_3class_svm_63%_20260509_2158.joblib (12.2MB)
数据: 厌恶725张 / 悲伤1514张 / 中性2486张 (共4725张, 板子热力图)
复用: V16的PCA50+Scaler (保证两级级联变换一致)
```

#### E2: 三类专用RF
```
总体(仅3类): 64.2%
  厌恶:  6.9% ❌ (RF对厌恶更差!)
  悲伤: 42.6% (+8pp)
  中性: 94.2% (+7pp)
```

**结论**: 三类专用**SVM**效果最好! 厌恶从17.5%→48.3%，悲伤从34.5%→49.8%

## 三、推荐方案：两级级联架构 ⭐⭐⭐

### 3.1 用户提出的核心思路

> "先来一个悲伤中性厌恶的判断，然后运行板子的逻辑。要是出现惊讶愤怒和开心就取代原来结果"

完全正确！这本质上是**前置过滤器 + 后端覆盖**的两级架构：

```
┌──────────────────────────────────────────────────────────┐
│                   输入: 128×128 热力图                    │
└────────────────────────┬─────────────────────────────────┘
                         │
                         ▼
┌──────────────────────────────────────────────────────────┐
│  第一级: 三类专用分类器 (厌恶 / 悲伤 / 中性)               │
│  ─────────────────────────────────────                    │
│  模型: 三类专用SVM + PCA50 (策略E)                        │
│  输出: pred_3 ∈ {厌恶, 悲伤, 中性}                        │
│  能力: 厌恶48% / 悲伤50% / 中性69%                         │
│                                                         │
│  ⚠️ 这一级可能误判其他4类(惊/恐/乐/怒)为这三类之一         │
└────────────────────────┬─────────────────────────────────┘
                         │ pred_3
                         ▼
┌──────────────────────────────────────────────────────────┐
│  第二级: 板子原有7类Ensemble                              │
│  ────────────────────────────                            │
│  模型: SVM + CNN + Rules (板子已有逻辑)                  │
│  输出: pred_7 ∈ {0惊喜,1恐惧,2厌恶,3快乐,4悲伤,5愤怒,6中性} │
│  强项: 惊喜63% / 恐惧14% / 快乐61% / 愤怒30% / 中性87%    │
└────────────────────────┬─────────────────────────────────┘
                         │ pred_7
                         ▼
┌──────────────────────────────────────────────────────────┐
│  融合决策层                                               │
│  ─────────                                              │
│                                                         │
│  IF pred_7 ∈ {惊喜, 恐惧, 快乐, 愤怒}                     │
│      → 直接输出 pred_7  ★ 板子强的类别优先               │
│                                                         │
│  ELSE (pred_7 ∈ {厌恶, 悲伤, 中性})                       │
│      → 输出 pred_3  ★ 用第一级的专门判断                 │
│                                                         │
│  特殊处理:                                               │
│  - 如果 pred_3 == 厌恶 且置信度低 → 降级为中性            │
│  - 如果 pred_3 == 悲伤 且置信度低 → 降级为中性            │
│  ("没到那个程度" 兜底策略)                                │
└────────────────────────┬─────────────────────────────────┘
                         │
                         ▼
                   最终预测结果
```

### 3.2 预期效果分析

| 类别 | 板子原逻辑(SVM) | 第一级专用SVM | 融合后预期 | 提升 |
|------|----------------|-------------|-----------|------|
| **惊喜** | 62.9% | N/A | **62.9%** | 不变 |
| **恐惧** | 13.8% | N/A | **13.8%** | 不变 |
| **厌恶** | 9.0% | **46.2%** | **35~46%** | **+26~37pp** 🚀 |
| **快乐** | 60.7% | N/A | **60.7%** | 不变 |
| **悲伤** | 35.0% | **54.5%** | **42~54%** | **+7~19pp** 🚀 |
| **愤怒** | 30.4% | N/A | **30.4%** | 不变 |
| **中性** | 87.1% | 72.8% | **73~85%** | -2~-14pp |
| **全体** | **56.0%** | — | **58~61%** | **+2~5pp** |

### 3.3 关键设计决策

**Q1: 为什么不直接替换板子逻辑？**
- A: 因为板子在{惊喜,快乐,愤怒}上表现已经不错(61-63%)，没必要冒险
- 只在弱项{厌恶,悲伤,中性}上引入新的专门判断

**Q2: 如果第一级把"惊喜"误判成"悲伤"怎么办？**
- A: 第二级会输出"惊喜"→ 融合层直接取"惊喜"→ 不会出错
- 只有当**两级都认为属于{厌恶,悲伤,中性}时**才用第一级结果

**Q3: 厌恶/悲伤判成中性可以接受吗？**
- A: 完全可以。用户明确说了"没到厌恶/悲伤的程度"
- 这比反过来(中性判成厌恶/悲伤)要好得多——前者是保守错，后者是激进错

**Q4: 第一级的训练数据怎么来？**
- A: 从已有的8050张中提取{厌恶,悲伤,中性}子集 = ~1367张
- 用同样的PCA50流程训练，不需要重新生成热力图

## 四、可选进阶优化（按投入产出比排序）

### 优化1: 结合策略D的投票规则 ⭐ 投入小收益大
```
在融合决策层加入RF辅助:
  IF pred_3 == 悲伤 AND rf_pred == 悲伤:
      → 高置信度悲伤 (双重确认)
  IF pred_3 == 中性 AND rf_pred == 悲伤 AND svm_pred != 悲伤:
      → 改判为悲伤 (利用RF对悲伤的敏感性)
```
预期: 悲伤再+5~10pp

### 优化2: 引入手工特征作为第三信号源
- 从原始JPG提取14维手工特征(眉距/嘴角/眼开度等)
- 训练一个轻量规则系统
- 与热力图SVM做late fusion
- **难点**: 需要在PC上跑MediaPipe或人脸关键点检测

### 优化3: 补全1169张缺失热力图
- 当前只用8050张，还有~14%数据没用
- 补完后重训所有模型
- 预期整体+1~3%

### 优化4: 数据增强（针对弱势类别）
- 厌恶只726张，且准确率最低
- 对厌恶样本做翻转/旋转/噪声增强
- 可能提升厌恶召回率

### 优化5: 更强的三类专用模型
- 试XGBoost/LightGBM（树模型在结构化数据上通常更好）
- 试神经网络MLP(但要注意过拟合风险)
- 试不同的PCA维度(30/70/100)对比

## 六、风险与兜底

| 风险 | 概率 | 影响 | 兜底方案 |
|------|------|------|---------|
| 三类专用SVM在板子上过拟合 | 中 | 中 | 用更大正则化(C<5) |
| PCA变换在板子上数值不一致 | 低 | 高 | 固定随机种子+保存完整pipeline |
| 推理速度超标(<33ms/帧) | 低 | 低 | SVM推理0.3ms, 完全够用 |
| 效果反而变差 | 低 | 中 | 保留回退路径：直接用板子原有逻辑 |
