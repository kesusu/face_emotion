### 2. 方向性错误: 对热力图特性的误判

| 错误认知 | 实际情况 | 证据 |
|---------|---------|------|
| "热力图保留空间结构,CNN能自动学习模式" | 热力图极度平滑, 无边缘/纹理可学 | 类内cos_sim=**0.97**, 类间=**0.965** |
| "480K参数的DeepCNN能学到细微差异" | 参数太多→过拟合严重 | TrainAcc 67% vs ValAcc 37%, gap=30% |
| "128×128分辨率比64×64更好" | 高分辨让弱类特征更难捕捉 | 中性从43.6%降到31.7%, 惊讶从38.5%降到26% |
| "调参能突破40%天花板" | 8轮实验证明CNN路线天花板≈42% | 38→40→37.9→35→37.5→40→39.9→37.2 |

### 3. 数据分析不足就动手

**应该先做的分析 (今天才做)**:
```python
# ① 热力图相似度分析
类内余弦相似度: 0.97~0.98   ← 极高! 图与图之间差别很小
类间余弦相似度: 0.965        ← 和类内只差0.01!
差距/噪声比 ≈ 0.01/0.05 = 非常低 → 这是高难度分类任务

# ② PCA信息密度
前10成分: 98.87%方差     ← 信息高度集中!
前20成分: 99.85%方差
前50成分: ~100%方差      ← 49152维→50维, 压缩比983:1

结论: CNN用49152维输入学480K参数 = 在稀疏信号上过拟合
正确做法: PCA降维→传统ML模型
```

## 二、热力图数据的本质特性

### 2.1 数据是什么?

```
每张热力图 = 468个MediaPipe人脸关键点的3通道高斯投影
Channel 0: XY位置高斯叠加    (哪里有点,哪里亮)
Channel 1: Z深度加权         (凸出面更亮)
Channel 2: 区域分组权重      (眉毛/眼睛/鼻子/嘴巴 不同亮度)

本质: 这是一个"人脸形状的模糊轮廓图", 不是照片!
```

### 2.2 关键统计

| 特性 | 值 | 含义 |
|------|-----|------|
| 维度 | (9906, 3, 128, 128) | 原始像素49152维 |
| 值域 | [0, 1] | 归一化高斯响应 |
| 全局均值 | 0.33 | 整体偏暗(大部分区域没有关键点) |
| 全局标准差 | 0.25 | 有一定动态范围 |
| **类内cos_sim** | **0.97-0.98** | **同类图像极度相似** |
| **类间cos_sim** | **0.965** | **异类图像也极度相似** |
| **类间-类内差距** | **~0.01** | **区分信号极其微弱** |

### 2.3 这意味着什么?

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  热力图之间的差异 ≈ 在一张均匀灰度图上加几个亮斑    │
│  这些亮斑的位置/强度差异只有1-2%                     │
│                                                     │
│  CNN擅长什么? 边缘、纹理、局部模式                    │
│  热力图有什么? 平滑的高斯团块, 没有边缘              │
│                                                     │
│  → CNN选错了! 就像用放大镜去测量温度                  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 5.2 短期优化 (本周)

#### 优化A: 解决RF的中性偏向问题

当前RF把90%的中性都判对了, 但其他类牺牲太大。

方案:
1. **代价敏感学习**: 不是简单balanced, 而是手动设权重(中性降低, 恐惧/厌恶升高)
2. **阈值调整**: 不取argmax, 而是对每类单独设阈值
3. **EasyEnsemble**: 对少数类过采样再训练多模型投票

#### 优化B: Ensemble融合

已知三个互补性强的模型:
- **SVM(14维手工特征)**: 54.5% → 多数类专家(开心82%, 中性70%)
- **RF(PCA50热力图)**: 47.2% → 空间模式专家
- **规则系统**: 27% → FACS先验知识, 惊讶/厌恶副标签强

融合预期: **55-60%**

#### 优化C: 5类合并方案 (用户之前提出的)

原7类 → 合并为5类:
- "激动" = 恐惧(332) + 愤怒(814) = 1146张
- "低落" = 悲伤(2141) + 中性部分
- 保留: 惊讶 / 厌恶 / 开心

减少类别数量直接提升每类的样本量和区分度。

## 七、附录: PCA成分的可解释性 (可选深入)

```
前5个主成分可能对应:
PC1 (~70%方差): 人脸整体位置/尺寸变化 (全局亮度)
PC2 (~12%方差): 表情强度强弱对比
PC3 (~6%方差):  上半脸(眉眼) vs 下半脸(嘴巴) 的激活比例
PC4 (~3%方差):  左右不对称程度
PC5 (~2%方差):  Z深度(前后)信息

→ 这说明情绪区分主要靠的是"整体形状"而非"局部纹理"
→ 再次印证CNN不是正确的选择!
```
