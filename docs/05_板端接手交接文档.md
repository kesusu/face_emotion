## 六、已知限制和经验教训

### 为什么CNN只有40%而不是更高?

1. **数据量瓶颈**: 4574样本 / 480K参数 = 105:1, 无论如何都会过拟合
2. **信息损失**: 468点→64×64高斯热力图丢失了FACS动作单元细节(AU4眉压等)
3. **数据集质量**: RAF-DB部分样本不符合FACS标准(悲伤不下垂, 愤怒像冷笑)
4. **但有ensemble价值**: CNN空间特征与SVM手工14维特征高度互补

### PC端踩过的坑 (不要再重复)

1. **SGD > AdamW**: 图像分类经典结论, SGD泛化好+1.7%
2. **Mixup不是万能药**: 提升难类但削弱易类, 总分不变
3. **Label Smoothing要轻微**: 0.05可以, 0.15直接欠拟合
4. **Dropout不能太大**: 0.5过正则, 0.4刚好
5. **数据增强要有**: 旋转±15°比±5°好很多
6. **GPU确定性设置必须加**: deterministic=True避免噪声
7. **CosineAnnealingLR比WarmRestarts稳定**

### 板端注意事项

1. CNN推理在CPU上即可 (<50ms), 不需要GPU版PyTorch
2. 热力图生成依赖scipy(ndimage.affine_transform)
3. ensemble模式下总延迟增加约30-50ms (仍在可接受范围)
