## 三、卡在哪

### 3.1 CNN的天花板 ≈ 48%

7次训练的完整轨迹：

```
V19a: 8.68%  ← 过度正则化，根本学不动
V19b: (中断)
V19c: 20.0%  ← 双重类别平衡，厌恶偏83%
V19d: 41.4%  ← 单Sampler太极端，中性偏81%
V19e: 47.6%  ← 无平衡，小类崩(恐惧0%)
V19f: 47.9%  ← 温和平衡，当前最佳 ★
V19g: (未跑完)

趋势: 8.68% → 47.9%，已趋于收敛
```

**为什么卡在48%**：
- 热力图类间cos_sim差仅0.005，CNN卷积核找不到有效的边缘/纹理
- 49152维输入中有效信号只占前20维，CNN参数浪费在冗余维度上
- 这不是调参问题，是模型归纳偏置与数据特性不匹配

### 3.2 弱类是终极瓶颈

7次训练中各类准确率的稳定模式：

| 类别 | V19e | V19f | 稳定性 | 诊断 |
|------|------|------|--------|------|
| 中性 | 88.6% | 83.6% | 稳定高 | 样本多(3074)，热力图明显 |
| 愤怒 | 41.1% | 50.3% | 中等 | 有提升空间 |
| 惊讶 | 46.3% | 50.5% | 中等 | 有提升空间 |
| 开心 | 45.5% | 39.6% | 波动 | 需要更好平衡 |
| 厌恶 | 0.6% | 19.7% | 极不稳定 | 小样本(862)，CNN弱点 |
| 悲伤 | 19.4% | 16.1% | 稳定低 | ★ 热力图和中性太像 |
| 恐惧 | 0% | 6.1% | 极低 | ★ 样本太少(332) |

**悲伤是CNN的盲区** —— 无论怎么调，悲伤始终<20%，因为悲伤热力图和中性几乎无法区分。

### 3.3 Expert v5.1的天花板 ≈ 65%

板端grid search结果：无论怎么调路由权重，总Acc卡在64.5-65%之间。
原因：组成Expert的三个子模型都有上限。

```
Expert = SVM7路由 + CNN强类 + SVM3弱类
                ↑           ↑
           59%总Acc     CNN在悲伤4%
                        → 悲伤永远拖后腿
```

## 四、为什么一直跑CNN没有大突破

### 4.1 做了什么

从V19a到V19g，系统性调了这些参数：
- Dropout: 0.65 → 0.6 → 0.5
- Weight Decay: 5e-3 → 1e-4
- Label Smoothing: 0.15 → 0.1
- 类别平衡: 双重 → 单Sampler → 无 → sqrt权重
- 损失函数: FocalLoss(alpha+gamma) → CE → FocalLoss(gamma only)
- 学习率: 5e-4 → 1e-3
- Scheduler: OneCycleLR → CosineAnnealing

### 4.2 边际收益递减

```
V19a→V19c: +11.3pp (去过度正则化，大步提升)
V19c→V19d: +21.4pp (去双重平衡，大步提升)
V19d→V19e: +6.2pp  (去单Sampler，中等提升)
V19e→V19f: +0.3pp  (温和平衡，几乎无提升)
V19f→V19g: ???     (降dropout+SWA，预期+1-2%)
```

**规律明确**: 从30%→48%靠"减法"(去正则/去多余平衡)，48%之后加法(SWA/更小dropout)的边际收益极小。

### 4.3 根本问题不在参数

```
┌──────────────────────────────────────┐
│                                      │
│  CNN + 热力图 = 工具选错             │
│                                      │
│  就像用放大镜量温度                   │
│  放大镜(CNN)很擅长看细节(边缘/纹理)  │
│  但温度(热力图)是全局分布信号         │
│  换多少倍的放大镜都量不出温度         │
│                                      │
│  → 正确做法: 换温度计(PCA+SVM/RF)    │
│                                      │
└──────────────────────────────────────┘
```

### 5.2 P0详解：为什么3类专用CNN ROI最高

**现状**: CNN对悲伤16.1%、中性83.6%、厌恶19.7%。
**问题**: 7类混在一起，悲伤和中性在CNN眼里几乎一样。

```
3类专用CNN:
  输入: 只取 厌恶(862)+悲伤(2141)+中性(3074) = 6077张
  任务: 3分类，不是7分类
  预期:
    悲伤: 16% → 35-40%  ← 专门学"悲伤vs中性"的差异
    中性: 83% → 85%     ← 保持
    厌恶: 20% → 40-50%  ← 样本足够，3类更容易学
```

**为什么有效**: 
- 7类时，悲伤和中性被"惊讶/恐惧/开心/愤怒"拉走了注意力
- 3类时，模型只需要区分3种模式，注意力集中
- 这和V17的3类SVM=54.5%的逻辑完全一致，但CNN在3类上可能更好

## 七、关键经验总结（一句话版）

1. **先分析数据特性再选模型** —— cos_sim差0.005就不该用CNN
2. **类别平衡只用一种** —— 双重平衡矫枉过正(V19c=20%)
3. **温和权重比极端权重好** —— sqrt(1/count) > 1/count
4. **弱类专用模型比全局调参有效** —— SVM3=54.5% vs 全局CNN=47.9%
5. **集成互补是突破天花板的关键** —— Expert v5=65% vs 单模型48%
6. **边际收益递减时停止调参** —— V19e→V19f仅+0.3%，该换方向了
7. **数据质量 > 模型复杂度** —— 脏数据清洗后效果可能比调参更好
