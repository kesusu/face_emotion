## 二、方案B(128×128 CNN)详细结果

### 2.1 训练配置

```
数据源:   RAF-DB/train 全量10330张 → MediaPipe成功8962张 (86.8%)
分辨率:  128×128 (高斯热力图, 3通道)
模型:    DeepCNN (480K参数, 4层卷积 + 2层全连接)
优化器:  SGD + Nesterov (momentum=0.9)
超参:    lr=0.003, batch=128, dropout=0.4, epochs=120
硬件:    RTX 4060 Laptop GPU (8GB), PyTorch 2.4.1+cu124
训练时间: ~20分钟
最佳点:  Epoch 99, ValAcc=37.8%
```

### 2.2 逐类准确率 vs 64×64旧版

| 类别 | **128新** | 旧64 | 变化 | 分析 |
|------|----------|------|------|------|
| 开心 | **68.4%** | 45.5% | **+22.9% ★★★** | 高分辨率保留嘴部/眉部细节 |
| 厌恶 | **58.3%** | 24.9% | **+33.4% ★★★** | 鼻子/唇部细节关键 |
| 愤怒 | **60.0%** | 50.3% | **+9.7% ★★** | 眉间/眼白细节提升 |
| 恐惧 | 33.3% | 28.8% | +4.5% | 略有提升 |
| 中性 | 31.7% | 43.6% | **-11.9% ↓↓** | 样本暴增(29.4%)引入噪声 |
| 惊讶 | 26.0% | 38.5% | **-12.5% ↓↓** | 微妙特征被高分辨率淹没 |
| 悲伤 | 20.6% | 23.7% | -3.1% ↓ | 八字眉信号变模糊 |

### 2.3 为什么加了数据和分辨率反而降了？

1. **过拟合加剧**: 参数480K vs 样本8962 = 仅19:1比例, 网络容量远超数据支撑能力
2. **中性样本污染**: 从666张暴增到2635张(占29.4%), RAF-DB的"中性"标签本身不可靠
3. **悲伤/惊讶特征被淹没**: 微妙的八字眉/圆睁眼在高分辨下反而更难捕捉

### 2.4 结论：方案B的新模型不如旧的！

| 版本 | 数据量 | 分辨率 | ValAcc | 推荐 |
|------|--------|--------|--------|------|
| **V9.5 旧(64×64)** | 4,574 | 64×64 | **40.0%** | ✅ **继续用这个** |
| V9.5 方案B(128×128) | 8,962 | 128×128 | 37.8% | 仅作参考/Ensemble备选 |

**纯CNN热力图路线天花板 ≈ 42-45%**, 8轮实验已证明: 38→40→37.9→35→37.5→40.0→39.9→37.2, **40%就是顶**

```python
def ensemble_predict(svm_result, cnn_result, rule_result):
    """
    三模型加权融合
    svm_result: {'label': str, 'confidence': float, 'probs': dict}
    cnn_result: {'label': str, 'confidence': float, 'probs': dict}
    rule_result: {'label': str, 'score': float, 'scores': dict}
    
    返回:
    {
        'primary': {'label': str, 'confidence': float},
        'secondary': {'label': str, 'confidence': float},
        'all_scores': dict,
        'interpretation': str
    }
    """
    WEIGHT_SVM = 0.40
    WEIGHT_CNN = 0.35
    WEIGHT_RULE = 0.25
    
    LABELS = ['惊讶','恐惧','厌恶','开心','悲伤','愤怒','中性']
    
    # 归一化各模型得分到概率分布
    def softmax(scores):
        import numpy as np
        vals = np.array([scores.get(l, 0) for l in LABELS], dtype=float)
        e = np.exp(vals - vals.max())
        return dict(zip(LABELS, e / e.sum()))
    
    svm_probs = softmax(svm_result.get('probs', {}))
    cnn_probs = softmax(cnn_result.get('probs', {}))
    rule_probs = softmax(rule_result.get('scores', {}))
    
    # 加权融合
    fused = {}
    for l in LABELS:
        fused[l] = WEIGHT_SVM * svm_probs.get(l, 0) + \
                   WEIGHT_CNN * cnn_probs.get(l, 0) + \
                   WEIGHT_RULE * rule_probs.get(l, 0)
    
    # Top-K排序
    ranked = sorted(fused.items(), key=lambda x: x[1], reverse=True)
    
    return {
        'primary': {'label': ranked[0][0], 'confidence': round(ranked[0][1], 4)},
        'secondary': {'label': ranked[1][0], 'confidence': round(ranked[1][1], 4)},
        'all_scores': {k: round(v, 4) for k, v in fused.items()},
    }
```

### 相比旧v8的改进点

1. **class_weight='balanced'** — 解决多数类偏向(开心/中性不再占优)
2. **StandardScaler标准化** — 特征归一化到零均值单位方差
3. **C=1.0线性核** — 通过3折CV选择最优超参
4. **probability=True** — 支持predict_proba()软概率输出(Ensemble必需)
5. **打包scaler** — 推理时自动标准化, 无需额外处理
