## §1 — 15_PC端训练任务书_V19_20260511.md

> Expert v5架构(64.6%)、DeepEmotionCNN模型代码、全7类+弱类3类CNN重训

```
输入人脸 → MediaPipe(468点)
  → SVM7路由(判断"强类or弱类")
    ├── 强类分支: CNN(7类) → Top-2输出
    │       恐惧≈88% / 惊讶≈85% / 愤怒≈82% / 开心≈75%
    │       (老CNN模型, 数据污染前训练的)
    └── 弱类分支: SVM3(3类专用) + 规则级联
            厌恶≈67% / 悲伤≈50% / 中性≈55%
整体 Ensemble: ~64.6%
```

### 1.3 任务2: 弱类3类CNN

## §2 — 15_两级级联策略_厌恶悲伤中性.md

> 三类混淆诊断、5种策略、两级级联架构设计

### 2.1 问题诊断
三大问题类：
- **厌恶**: SVM(17%)+规则(17%) → Ensemble(37%)
- **悲伤**: SVM(27%)+规则(10%) → Ensemble(22%)
- **中性**: SVM(42%)+规则(20%) → Ensemble(31%)

### 2.2 5种策略实验
| 策略 | 方案 | 预期 |
|------|------|------|
| E1: 权重搜索 | CNN×2.0+SVM×0.3+规则×0.3 | 72% |
| E2: 类专用模型 | 每类一个专用模型 | 58-61% |
| **E3: 置信度门控** | CNN p>0.7用CNN, 否则用SVM | — |
| E4: 级联分类 | SVM7路由+专用模型 | 65-70% |
| E5: 两级级联 | 前置过滤器+后端覆盖 | **58-61%** (厌恶) |

### 2.3 两级级联架构（推荐）
```
第一级: 前置过滤器（区分"强类"和"弱类"）
  - SVM7路由 → 置信度>0.5走强类分支
  - 强类: CNN直接输出
  - 弱类: 进入第二级

第二级: 后端覆盖（弱类专用处理）
  - 厌恶分支: SVM3级联
  - 悲伤分支: 规则+阈值
  - 中性分支: 规则兜底
```

## §3 — 16_V19_CNN训练记录.md

> V19a→g共7次训练尝试、中性偏执问题、V19f最佳47.9%

### 3.1 训练历程
| 版本 | ValAcc | 说明 |
|------|--------|------|
| V19a | 30-36% | 小网络尝试，架构问题 |
| V19b | 39.7% | 恢复DeepEmotionCNN |
| V19c | 43.0% | dropout=0.5, 128×128 |
| V19d | 44.0% | 优化配置 |
| V19e | 45.8% | Fine-tune |
| **V19f** | **47.9%** | **★ dropout=0.6, ep66** |
| V19g-a | 40.2% | 中性偏执修复版 |

### 3.2 V19f 最佳模型
| 指标 | 值 |
|------|-----|
| ValAcc | **47.9%** |
| 中性recall | 83.6% |
| 非中性→中性 | 56%（严重偏执） |
| 数据 | 9906张, 128×128 |
| 配置 | dropout=0.6, SGD+Nesterov, LR=0.01, wd=1e-4 |
| 模型大小 | 1.9MB (部署版, 仅权重) |

### 3.3 类别平衡四大陷阱
```
陷阱1: 加权Loss不是万能 — 恐惧(332张)权重×3还是学不好
陷阱2: 过采样会导致过拟合 — 小类反复见过拟合
陷阱3: 欠采样会丢失信息 — 大类下采样损失数据
陷阱4: 中性偏执的副作用 — 压中性同时也会压低其他类
```

### 3.4 V19g-a 中性偏执修复
- 方案: alpha中性=0.5 + 截断中性到1500张
- 结果: ValAcc从40.2%→最终46.2%(V19g-b)/46.7%(V19g-c)
- 中性recall: 83.6%→65.4%/74.3%
- 非中性→中性: 56%→37.6%/46.8%

## §4 — 17_全面复盘_项目进度与方向.md

> 从30.6%到64.9%的突破路径、CNN天花板48%分析

### 4.1 突破路径
```
V15 CNN(30.6%) → 发现CNN在热力图上的天花板
     → RF(PCA50, 47.2%) → 转折: ML > CNN
     → SVM(14维, 54.5%) → 验证手工特征天花板
     → V19 CNN(47.9%) → 复现CNN天花板
     → Expert v5.1(64.9%) → 当前最佳
```

### 4.2 CNN天花板分析
| 模型 | ValAcc | 说明 |
|------|--------|------|
| V15 boardSDK | 30.6% | 深网络 |
| V16 RF(PCA50) | **47.2%** | ★ 简单模型更好 |
| V19f | 47.9% | 简单网络+dropout |
| V19g-a | 40.2% | 中性偏执修复 |

**结论**: CNN在热力图上的天花板约48% → 非架构问题，是数据限制（恐惧仅332张）

### 4.3 7条关键经验
1. **数据先行** — 先分析数据特性再选模型
2. **简单模型未必差** — RF(PCA50) > CNN(480K参数)
3. **正交性验证** — 三模型错误模式不同才有融合价值
4. **不要CNN万能** — CNN不适合平滑高斯热力图
5. **中性偏执难解** — 压中性=压弱类，必须温和
6. **弱类需要专用模型** — 全局模型照顾不了少数类
7. **板端实际效果更重要** — PC ValAcc 48% ≠ 板端集成表现

### 4.4 P0方案
| 优先级 | 方案 | 预期 |
|--------|------|------|
| P0 | 3类弱类专用CNN | +5-10pp |
| P0 | 时序平滑(3-5帧) | +3-5pp |
| P1 | 补充弱类数据 | +5-10pp |
| P1 | 可疑样本清洗 | +1-3pp |
