| 指标 | 值 |

### 2.2 根因

1. **训练数据含脏数据** — train2(9906张) vs new_train(9207张, 清洗后), 差异约700张
2. **类别不平衡** — 中性3074张 vs 恐惧332张 (9:1)
3. **数据增强不足** — CNN在小数据上容易记住训练集
4. **网络可能偏大** — 对于9906张128x128的数据量

#### 模型架构 (必须与板端一致!)
```python
import torch.nn as nn

class ResidualBlock(nn.Module):
    def __init__(self, c):
        super().__init__()
        self.conv = nn.Sequential(
            nn.Conv2d(c, c, 3, padding=1), nn.BatchNorm2d(c),
            nn.ReLU(inplace=True),
            nn.Conv2d(c, c, 3, padding=1), nn.BatchNorm2d(c))
        self.relu = nn.ReLU(inplace=True)
    def forward(self, x): return self.relu(x + self.conv(x))

class DeepEmotionCNN(nn.Module):
    def __init__(self, nc=7, dr=0.6):
        super().__init__()
        self.stem = nn.Sequential(nn.Conv2d(3,24,3,padding=1), nn.BatchNorm2d(24),
            nn.ReLU(inplace=True), nn.MaxPool2d(2))
        self.stage1 = nn.Sequential(nn.Conv2d(24,32,3,padding=1), nn.BatchNorm2d(32),
            nn.ReLU(inplace=True), ResidualBlock(32), nn.MaxPool2d(2))
        self.stage2 = nn.Sequential(nn.Conv2d(32,48,3,padding=1), nn.BatchNorm2d(48),
            nn.ReLU(inplace=True), ResidualBlock(48), nn.MaxPool2d(2))
        self.stage3 = nn.Sequential(nn.Conv2d(48,64,3,padding=1), nn.BatchNorm2d(64),
            nn.ReLU(inplace=True), ResidualBlock(64), nn.AdaptiveAvgPool2d(4))
        self.classifier = nn.Sequential(
            nn.Flatten(), nn.Linear(64*4*4, 256), nn.BatchNorm1d(256),
            nn.ReLU(inplace=True), nn.Dropout(dr),
            nn.Linear(256, 128), nn.ReLU(inplace=True),
            nn.Dropout(dr*0.5), nn.Linear(128, nc))
    def forward(self, x):
        return self.classifier(self.stage3(self.stage2(self.stage1(self.stem(x)))))
```

**⚠️ 注意层名**: 必须用 `stem/stage1/stage2/stage3/classifier`，不要用 `s1/s2/s3/cls`
(之前踩过这个坑，导致模型在板端输出全0)

#### 训练建议

```python
# 1. 数据划分: 80% train / 20% val
#    建议: 按类别分层抽样, 确保验证集各类都有足够样本

# 2. 数据增强 (重要!):
#    - RandomHorizontalFlip (p=0.5)
#    - RandomRotation (±10°)
#    - 随机亮度/对比度微调
#    - RandomErasing (p=0.1) — 防止过拟合

# 3. 类别平衡:
#    方案A: WeightedRandomSampler (按类别权重采样)
#    方案B: 每类固定采样数量 (min_class_size * 2)
#    方案C: Focal Loss (关注难样本)
#    → 推荐方案A+数据增强组合

# 4. 优化器: AdamW, lr=1e-3, weight_decay=1e-4
# 5. Scheduler: CosineAnnealingLR, T_max=50
# 6. Batch size: 32-64 (看GPU显存)
# 7. Epochs: 80-150 (用early stopping, patience=15)

# 8. 正则化:
#    - Dropout: 0.6 (当前值, 可以试试0.5-0.7)

```

## 七、关键经验教训 (避免踩坑)

| 教训 | 说明 |
|------|------|
| ❌ 层名必须一致 | `stem/stage1/stage2/stage3/classifier` 不是 `s1/s2/s3/cls` |
| ❌ 不要用PC MediaPipe的热力图 | 必须用板端SDK生成的, 否则CNN部署后会崩 |
| ❌ 概率尺度不可比 | CNN softmax和SVM概率不能直接加权, 要用离散路由 |
| ❌ CNN不能做路由器 | CNN几乎不预测悲伤/中性, 用CNN输出路由会导致弱类全崩 |
| ✅ SVM7做路由器最稳 | SVM7对7个类都有输出, 适合做路由判断 |
| ✅ 强类听CNN,弱类听级联 | CNN强类74-88%, 级联弱类50-66%, 互补极好 |
| ✅ 板端已验证阈值天花板 | 调参只给+0.3pp, CNN质量才是关键 |
